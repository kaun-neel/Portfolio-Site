NASA Exoplanet Habitability Scorer

[ download pdf ↓ ] [ source code ↗ ]

SQL Data Engineering Case Study

1. Executive Summary

This project presents a fully functional Exoplanet Habitability Scoring Engine built entirely in SQL, using real observational data sourced from NASA's Planetary Systems Composite Parameters archive. The system ingests data for 6,138 confirmed exoplanets across 4,610 host star systems and applies a scientifically grounded, multi-factor weighted algorithm to rank each planet by its potential to support life.

The scoring model evaluates five independent dimensions: planetary size, equilibrium temperature, orbital position within the stellar habitable zone, host star stability, and proximity to Earth, thus assigning a maximum of 100 points per planet. Results are classified into four tiers: Highly Habitable, Potentially Habitable, Marginally Habitable, and Uninhabitable.

Key Findings at a Glance

  • 6,138 confirmed exoplanets analysed from the NASA Planetary Archive
  • 4,610 unique host stars with computed habitable zone boundaries
  • 76.7% of planets classified as Uninhabitable based on the 5-factor model
  • 22.0% classified as Marginally Habitable showing partial life-supporting conditions
  • A small but significant subset identified as Potentially or Highly Habitable
  • Top-ranked planets score across all five dimensions simultaneously

2. Project Overview

2.1 Motivation

The search for habitable worlds beyond Earth is one of the defining scientific endeavours of our time. NASA's Kepler, TESS, and James Webb Space Telescope missions have collectively confirmed thousands of exoplanets, each characterised by dozens of physical measurements. Yet translating raw astronomical data into a meaningful habitability ranking requires structured data engineering and precisely the domain of SQL.

This project was conceived to demonstrate that SQL is not merely a query language for business reporting, but a powerful analytical tool capable of implementing scientific algorithms. By building this engine on real NASA data, the project bridges two disciplines: astronomy and data engineering.

2.2 Objectives

  • Design a normalised relational database schema for planetary and stellar data
  • Import and clean 6,138+ rows of real NASA observational data
  • Derive missing columns (stellar luminosity, orbital semi-major axis) from physics equations
  • Implement a 5-factor weighted habitability scoring algorithm in pure SQL
  • Rank all planets using SQL window functions and classify them into habitability tiers
  • Visualise findings with five professional Python charts

3.2 Derived Columns

The raw NASA dataset was missing several columns required by the scoring model. Rather than discarding affected rows, two critical values were mathematically derived using established physics equations, demonstrating domain knowledge beyond SQL alone.

Derived Column Formula Applied Scientific Basis
st_luminosity L = R^2 x (T / 5778)^4 Stefan-Boltzmann Law
pl_orbsmax a = (T_yr^2 x M_star)^(1/3) Kepler's Third Law

4. Database Schema Design

The database follows third normal form (3NF) to eliminate data redundancy. Star attributes are stored once in the star_systems table and referenced by all associated planets via a foreign key, mirroring industry-standard data warehouse design patterns.

4.1 Tables & Relationships

  • star_systems (4,610 rows): Stores host star measurements and derived luminosity
  • exoplanets (6,138 rows): Stores all planetary measurements with FK to star
  • habitable_zone (3,713 rows): Pre-computed HZ boundaries per star (inner/outer AU)
  • habitability_scores (—): Stores final scores and classification per planet

Relationships:
• star_systems (1) ──> exoplanets (many) via star_id
• star_systems (1) ──> habitable_zone (1) via star_id
• exoplanets (1) ──> habitability_scores (1) via planet_id

5. The Habitability Scoring Algorithm

The scoring model assigns up to 100 points per planet across five independent dimensions. Each dimension is implemented as a CASE expression within a layered CTE structure, allowing each factor to be audited, adjusted, and extended independently.

Dimension Max Scoring Criteria Rationale
Planet Size 20 pts 0.8–1.5 RE = 20 pts; 1.5–2.0 RE = 14 pts; 0.5–0.8 RE = 10 pts Rocky Earth-like planets most likely to hold atmosphere & liquid water
Temperature 25 pts 260–320 K = 25 pts; 230–260 K = 18 pts; 320–370 K = 15 pts Liquid water requires 273–373 K; sweet spot maximised near 290 K
Habitable Zone 25 pts Inside HZ = 25 pts; Within 15% of HZ edge = 12 pts Orbital position determines long-term stellar energy flux balance
Star Stability 20 pts G/K type (4000–7000 K, 0.5–1.4 solar masses) = 20 pts Sun-like stars offer stable UV environment and multi-billion year lifespan
Distance 10 pts <20 pc = 10 pts; <100 pc = 7 pts; <500 pc = 4 pts Proximity enables spectroscopic follow up and future mission planning

6. SQL Techniques Demonstrated

A multi-layer CTE pipeline was used to simplify complex calculations. Instead of writing deeply nested queries, the logic was divided into sequential steps (HZ calculation → scoring → totals → ranking).

Conditional scoring logic was implemented using CASE expressions, and Window functions such as RANK() OVER() were used to generate a global leaderboard. Derived limits were inserted iteratively via INSERT ... SELECT statements while mathematical functions extracted exact HZ boundaries natively within the RDMS engine.

7. Results & Findings

Fig 4: Habitability Distribution Pie Chart

7.1 Habitability Distribution

After running the full scoring engine across all 6,138 planets with complete data, the results reveal a distribution consistent with scientific expectation — the vast majority of known exoplanets are hostile to life as we know it, while a small but meaningful fraction show promising characteristics.

  • 76.7% — Uninhabitable: extreme temperatures, gas giants, or poor stellar environments
  • 22.0% — Marginally Habitable: meet 1–2 criteria, limited by temperature or orbit
  • ~1.1% — Potentially Habitable: strong multi-factor scores, promising candidates
  • < 0.2% — Highly Habitable: score 80+ across all five dimensions simultaneously

7.2 What Makes a Top-Ranked Planet?

Analysis of the highest-scoring planets reveals a consistent profile. Top candidates are invariably Earth-sized (0.8–1.5 Earth radii), maintain equilibrium temperatures between 260–320 Kelvin, orbit comfortably within their star's habitable zone, and orbit G or K-type stars with masses between 0.5 and 1.4 solar masses — closely resembling the conditions of our Solar System.

High-scoring candidates cluster within 250 parsecs, making them accessible targets for JWST follow-up missions.
